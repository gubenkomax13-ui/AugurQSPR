# Как работает оптимизация гиперпараметров

## Почему этот блок находится перед обучением модели

Гиперпараметры зависят от выбранного алгоритма. Например, для Random Forest настраиваются число и глубина деревьев, а для SVR — `C`, `epsilon` и параметры ядра. Поэтому поиск имеет смысл только после выбора модели.

## Что происходит после включения

При нажатии кнопки обучения программа перебирает комбинации гиперпараметров и оценивает каждую комбинацию с помощью внутренней K-Fold кросс-валидации.

После поиска выбирается комбинация с лучшей CV-метрикой, и итоговая аналитическая модель обучается на всех доступных веществах с найденными параметрами.

## Grid Search и Random Search

**Grid Search** проверяет все комбинации заданной сетки. Он воспроизводим и хорошо подходит для небольшого пространства параметров, но может работать дольше.

**Random Search** проверяет случайную часть комбинаций. Он обычно быстрее для моделей с большим числом настраиваемых параметров.

## Число фолдов CV

- 5 фолдов — рекомендуемый старт;
- больше фолдов даёт более подробную оценку, но увеличивает время поиска;
- для небольших выборок число фолдов не должно быть слишком большим.

## Связь с отбором дескрипторов

Если автоматический отбор дескрипторов включён, во время оценки комбинаций он выполняется внутри обучающих фолдов. Предварительный отбор из верхнего блока используется только для диагностики до / после и не подменяет этот процесс.

Результат внутреннего поиска не является независимой внешней валидацией. Качество выбранной модели нужно отдельно оценивать в разделе валидации.

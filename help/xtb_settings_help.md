\# Справка по настройкам xTB



\## Метод xTB



\### GFN2-xTB

Самый современный и обычно самый точный вариант из трёх. Хороший выбор по умолчанию.



Рекомендуется, если важны адекватная 3D-геометрия, нековалентные взаимодействия, дипольный момент, HOMO/LUMO и общая надёжность расчёта.



\### GFN1-xTB

Более старый метод. Может быть быстрее или стабильнее для отдельных молекул, но обычно уступает GFN2-xTB по точности.



Имеет смысл пробовать, если GFN2-xTB не сходится или даёт много ошибок на конкретном наборе.



\### GFN0-xTB

Самый простой и быстрый вариант. Подходит для предварительного скрининга, очень больших наборов молекул или быстрой оценки, когда высокая точность не критична.



Для финальной QSPR-модели лучше использовать только после сравнения с GFN2-xTB.



\---



\## Заряд молекулы



Параметр \*\*«Заряд молекулы»\*\* задаёт общий формальный заряд системы.



Обычно:

\- для нейтральных органических молекул ставят `0`;

\- для катионов ставят `+1`, `+2` и т.д.;

\- для анионов ставят `-1`, `-2` и т.д.



Если в датасете все вещества нейтральные, оставляйте `0`.



Если в наборе есть соли, ионные формы или заряженные молекулы, лучше заранее стандартизировать структуры и решить, какая форма должна моделироваться: нейтральная, катионная, анионная или соль.



\---



\## UHF / число неспаренных электронов



Параметр \*\*UHF\*\* задаёт число неспаренных электронов.



Обычно:

\- для обычных закрытооболочечных органических молекул ставят `0`;

\- для радикалов ставят `1`;

\- для триплетных состояний или систем с двумя неспаренными электронами ставят `2`.



Для стандартных QSPR-задач по нейтральным органическим молекулам почти всегда оставляют `0`.



Менять стоит только если вы точно знаете, что вещество является радикалом или открытооболочечной системой.



\---



\## Accuracy



Параметр \*\*Accuracy\*\* управляет строгостью расчёта.



Рекомендуемые значения:

\- `1.0` — стандартный вариант, хороший баланс скорости и качества;

\- меньше `1.0` — более строгий расчёт, может быть медленнее;

\- больше `1.0` — менее строгий расчёт, может быть быстрее, но менее надёжен.



Для обычного применения оставляйте `1.0`.



Если расчёты нестабильны или есть сомнения в качестве, можно попробовать `0.5`.



Для быстрого чернового скрининга можно использовать `2.0–5.0`.



\---



\## Electronic temperature, K



\*\*Electronic temperature\*\* влияет на электронное заселение уровней.



Обычно оставляют `300 K`. Это стандартное значение для большинства органических молекул.



Повышать температуру имеет смысл редко: например, если есть проблемы со сходимостью для сложных систем.



Для обычных QSPR-расчётов менять этот параметр не нужно.



\---



\## Max iterations



\*\*Max iterations\*\* — максимальное число итераций самосогласованного расчёта.



Рекомендуемые значения:

\- `250` — нормальное значение по умолчанию;

\- `500` — если часть молекул не сходится;

\- `1000–2000` — для сложных случаев, но расчёт может стать заметно медленнее.



Если большинство молекул считается нормально, оставляйте `250`.



\---



\## Предварительная оптимизация 3D-геометрии RDKit



Перед xTB программа строит 3D-структуру из SMILES.



Опция \*\*«Предварительно оптимизировать 3D-геометрию RDKit»\*\* включает быструю MMFF/UFF-оптимизацию перед xTB.



Обычно лучше оставить включённой. Это помогает xTB стартовать с более разумной геометрии.



Отключать стоит только для диагностики или если RDKit-оптимизация сама вызывает проблемы.



\---



\## Ограничить число молекул для xTB



Этот параметр нужен для тестового запуска.



Рекомендуемый порядок:

1\. Сначала поставить `5` или `10` и проверить, что расчёт работает.

2\. Потом поставить `50–100` для оценки скорости.

3\. После проверки поставить `0`, чтобы считать весь датасет.



`0` означает: считать все молекулы.



\---



\## Практические рекомендации



\### Для первого теста

\- метод: `GFN2-xTB`;

\- заряд: `0`;

\- UHF: `0`;

\- Accuracy: `1.0`;

\- Electronic temperature: `300 K`;

\- Max iterations: `250`;

\- RDKit-оптимизация: включена;

\- ограничение молекул: `5` или `10`.



\### Для финального расчёта

\- метод: `GFN2-xTB`;

\- ограничение молекул: `0`;

\- остальные параметры оставить стандартными, если нет специальных причин их менять.



\### Для очень большого датасета

\- сначала проверить `GFN0-xTB` или ограничение на часть молекул;

\- затем сравнить качество модели с GFN2-xTB на небольшой подвыборке.

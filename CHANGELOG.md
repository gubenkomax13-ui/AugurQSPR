# Журнал исправлений

## 2026-07-18

- Устранено зависание после расчёта Mordred: проверка выбранных имён переведена с повторного поиска по списку на `set`, успешные результаты больше не создают сотни тысяч диагностических записей, ошибки хранятся ограниченной выборкой, временные объекты очищаются порциями, а широкая матрица не дублируется в `session_state`.
- Контроль утечки для уже числовой матрицы Mordred больше не преобразует каждый элемент через строки и использует прямой числовой путь без лишней полной копии данных.
- Исправлен сбой сравнения моделей: в `qspr_app.py` добавлен импорт `get_model_group`, чтобы сводная таблица рейтинга не падала с `NameError`.
- Методически разделены сценарии валидации: режим `Одна модель` открывает подробную валидацию/диагностику по выбранной модели, а режим `Обучить несколько моделей` остаётся в сравнительной валидации и не дублирует нижний блок валидации.
- После дескрипторов добавлен переключатель режима обучения: `Одна модель` или `Обучить несколько моделей`; в UI рендерится только выбранный сценарий, чтобы не грузить лишние блоки.
- После расчёта дескрипторов поток UI теперь рендерит обучение одной модели и сравнение/автообучение нескольких моделей, а валидация, диагностика, консенсус, прогноз и отчёты не рендерятся до появления обученной модели.
- Контроль утечки данных после сборки дескрипторов дополнительно ускорен: проверка near-identical и корреляций с целевым свойством переведена из цикла по колонкам в один векторный проход по матрице.
- Таблицы покрытия датасета дескрипторами и качества строк больше не рендерятся автоматически: в UI остаются метрики, сводка и CSV-выгрузка, а подробности открываются по чекбоксу.
- Для больших дескрипторных матриц расширенная диагностика дескрипторов больше не запускается автоматически сразу после контроля утечки: добавлены сообщение и кнопка ручного запуска, а расчёт корреляций переведён на быстрый векторный режим.
- После SAOD-очистки теперь обновляется `dataset_signature` и запись dataset в analysis bundle, чтобы следующий `rerun` не сбрасывал баннер и не возвращал состояние к исходной подписи датасета.
- Баннер о новом SAOD-очищенном датасете перед блоком дескрипторов вынесен в отдельное состояние `descriptor_preflight_banner`, чтобы он показывался сразу после `st.rerun()` и не зависел от сохранности SAOD-результатов.
- Добавлен устойчивый баннер перед блоком дескрипторов после ручной или автоматической SAOD-очистки: показывается, что дальше используется новый очищенный датасет, сколько веществ осталось и сколько исключено. Статус очистки закреплён на уровне датасета и не теряется при сбросе старых расчётов.
- Оптимизирован контроль утечки данных после расчёта дескрипторов: числовые признаки приводятся к numeric пачкой, результат кэшируется, короткие маркеры `bp`/`mp`/`exp` больше не срабатывают как подстроки внутри имён дескрипторов Mordred.
- Переименован инструмент в интерфейсе: `SAOD (Структурная проверка и контроль согласованности данных)`.
- Уточнено описание SAOD: модуль описан как анализ любых веществ, а не только алканов.
- Добавлен полный Windows-сценарий установки и запуска: `setup_full.bat`, `setup_full.ps1`, `run_augur.bat`, `run_augur.ps1`.
- Установщик научился проверять доступность 25 моделей-кандидатов и прогревать PySR/Juliacall.
- Исправлены русские подписи в SAOD-таблицах и перевод блока источников дескрипторов.
- Оптимизирован расчёт молекулярных дескрипторов:
  - RDKit оставлен самым лёгким режимом.
  - Mordred считается как `RDKit + уникальные Mordred`, без лишнего расчёта полного Mordred-каталога.
  - Smart-режим считается как `RDKit + уникальные Mordred + уникальные PaDEL`.
  - PaDEL больше не запускает лишние fingerprint/1D-2D проходы, если выбранный список можно собрать одним проходом.
  - DScribe переиспользует Coulomb Matrix-калькулятор на весь датасет.
- Пересобран каталог дескрипторов под текущее окружение: RDKit 208, уникальные Mordred 1558, уникальные PaDEL 1238, полный PaDEL 2756.
